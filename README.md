CS 300 Portfolio Reflection
What problem were you solving in the projects for this course?

The main problem I was solving was creating a course advising program that could read course information from a file, store it in an efficient data structure, and allow users to quickly look up courses and their prerequisites. The program also needed to display all courses in alphanumeric order while handling invalid input and missing data.

How did you approach the problem? Why are data structures important?

Before writing any code, I compared different data structures to decide which one would work best. I looked at vectors, hash tables, and binary search trees. Even though hash tables are fast for searching, I chose a structure that made it easier to keep the courses in sorted order. Working through that process helped me understand that choosing the right data structure can make a program simpler, faster, and easier to maintain. Instead of just making the program work, I had to think about how efficiently it would work as more data was added.

How did you overcome any roadblocks?

One of the biggest challenges was reading the course file correctly and making sure the data was valid before storing it. I also had to make sure prerequisites existed and handle cases where users entered invalid course numbers. I worked through those problems by breaking the project into smaller pieces, testing each function as I finished it, and fixing issues one at a time instead of trying to solve everything at once.

How has this project changed the way you design software?

This project helped me realize that planning before coding saves a lot of time later. Creating pseudocode and thinking through the data structure first made the actual coding much easier. I also learned that designing software isn't just about getting the correct output—it's about building something that's organized, efficient, and easy for someone else to understand.

How has your programming style changed?

After completing this project, I pay more attention to writing clean and organized code. I try to give variables and functions meaningful names, separate different tasks into their own functions, and include error checking instead of assuming everything will work perfectly. Those habits make my programs easier to read, debug, and update if changes are needed later. I also have a better understanding of how choosing the right data structure can make future improvements much easier.
